# Multi-language setup

***

The Multi-Language Accounts Setup enables international companies to operate in different languages within a single set of books, facilitating communication with local investors and stakeholders. This feature allows users to customize the language interface and translate ledger accounts. Users can easily switch between languages or customize them according to their specific needs. Additionally, the plugin supports multi-language user interfaces and enables the presentation of financial data in local languages, enhancing accessibility and comprehension for diverse audiences.

\===

The multi-language ledgers, groups, types and journals (batches) makes it possible for an international company to work in different languages on the same set of books.

This helps to show local investors in their language the balances and turnover of the company.&nbsp;

The multi-language feature allows different users to use the same set of books in the user language interface. The translation of ledger accounts need to be purchased from the shop.&nbsp;

By default, each Set of Books will run in the selected language when launching osFinancials for the first time (new installation). You may, if necessary (i.e. incorrect language is selected) or when you need to work in a different language, use **Start → [Switch language**](<switch-language.md>) to change the language.

Any language may be customised / edited for your specific requirements on **Setup → Tools → Customise language**.

In addition to this, you may activate the Multi-Language feature (using any of the available languages in **Start → [Switch language**](<switch-language.md>)) in a Set of Books to:

* Create a Multi-Language User Interface. This makes it possible for a multi-lingual company to work in different languages on the same set of books.
* Create a Multi-Language Accounts (Chart of Accounts) - This helps to show local investors in their language the balances and turnover of the company.&nbsp;
