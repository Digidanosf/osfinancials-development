# MSSQL Server's backup and restore options

***

If you are running a Set of Books as a Microsoft SQL Server (MSSQL) database type, MSSQL Server includes its own options to backup and restore the database.

MSSQL Server provides built-in features and tools to perform backups and restores, ensuring the reliability and integrity of your database. These features include:

1. **SQL Server Management Studio (SSMS)**: SSMS is a graphical tool provided by Microsoft for managing MSSQL Server. It includes options to perform backups and restores through its user interface. You can access these options by connecting to your MSSQL Server instance using SSMS and navigating to the appropriate database.
1. **Transact-SQL Commands**: MSSQL Server supports Transact-SQL (T-SQL), a programming language used for managing and querying databases. T-SQL provides specific commands for performing backups and restores, such as **BACKUP DATABASE** and **RESTORE DATABASE**. These commands can be executed using an SQL client or through scripts.

By utilizing MSSQL Server's backup and restore options, you can create backups of your Set of Books database at regular intervals and restore them when needed. Backups can be stored locally or on remote storage devices for added data protection.

It's important to understand the backup and restore process specific to MSSQL Server and follow best practices to ensure the integrity and availability of your database. Consulting the MSSQL Server documentation or seeking guidance from database administrators familiar with MSSQL Server will provide you with detailed instructions and considerations for performing backups and restores.
