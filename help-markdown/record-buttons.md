# Record buttons

***

There are ten (10) record buttons are used to manage records some of the following screens:

* **Setup → Accounts** ([**Setup**](<setup-ribbon.md>) ribbon)
* **Stock items** (**Multi-group**) tab (**Default** ribbon)
* **Setup** **→ Stock information -** (**Tax classes**) tab ([**Setup**](<setup-ribbon.md>) ribbon)
* **Setup → Currencies** ([**Setup**](<setup-ribbon.md>) ribbon)

These are:

1. ![Image](<lib/record-icons-01.png>) **First record** - Go to the first record.
1. ![Image](<lib/record-icons-02.png>) **Prior record** - Go to the previous record.
1. ![Image](<lib/record-icons-03.png>) **Next record** - Go to the next record.
1. ![Image](<lib/record-icons-04.png>) **Last** **record** - Go to the last record.
1. ![Image](<lib/record-icons-05.png>) **Insert record** (add) a new record.
1. ![Image](<lib/record-icons-06.png>) **Delete record** - (permanently remove) a record. Once you click on this button, a confirmation message&nbsp; will be displayed:

*"Delete record?"*

| ![Image](<lib/admon-important.png>) | *Make absolutely sure that the correct record before you click on the **OK** button. If it is not the correct record, click on the **Cancel** button and select the correct record.* |
| --- | --- |


7. ![Image](<lib/record-icons-07.png>) **Edit record** - Edit a selected record. This will activate the **Post edit** or **Cancel edit** buttons.
7. ![Image](<lib/record-icons-08.png>) **Post edit -** Saves (accept) the changes to a record.&nbsp;
7. ![Image](<lib/record-icons-09.png>) **Cancel** **edit** - Cancels the changes to a record.
7. &nbsp;![Image](<lib/record-icons-10.png>)**Refresh data** - This will refresh the latest changes.
