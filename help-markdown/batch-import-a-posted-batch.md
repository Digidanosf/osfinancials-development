# Import a posted batch

***

This option allows you to import the transactions from a posted batch in the same Set of Books.

| ![Image](<lib/admon-tip.png>) | *If you need to import posted batches from TurboCASH4 or a different Set of Books in osFinancials, osFinancials5/TurboCASH5, you need to:* *On the **Setup** ribbon, go to **Export** (**Posted batches** tab) and create a valid export file in the Set of Books containing that batch.*&nbsp; *On the **Default** ribbon, go to **Batch entry - F9:Process** and select the "Posted batches import" option in the Set of Books you need to import the batch.*&nbsp; |
| --- | --- |


**To import a posted batch:**

1. On the **Default** ribbon, select **Batch entry** (**F2**).&nbsp;
1. Select a Batch type and click on the **Open** button.&nbsp;
1. Click on the **F9:Process** icon. The "*Process the batch*" options screen is displayed: &nbsp;

![Image](<lib/batch-process-import-a-posted-batch.png>)

4. Select the **Import a posted batch** option, and click on the **OK** button. The "*Reverse posted batch/document"* screen is displayed:

![Image](<lib/batch-process-import-a-posted-batch-select.png>)

5. All posted batches associated with the selected batch type, will be listed. Select a batch to import. The Alias (batch name) makes it easier to identify the correct batch to import. View the details of the batch to make absolutely sure that it is the correct batch (journal).&nbsp;

| ![Image](<lib/admon-tip.png>) | *To view the details of the transactions in the selected batch (journal), you may:-*&nbsp; *Double-click on the selected batch (journal).*&nbsp; *On the **Details** tab, the transactions for the selected batch will be listed. (If you wish to print a list of the transactions in the posted batch, you may click on the **Print** button to print a "Batch type - Posted batch" report).*&nbsp; |
| --- | --- |


6. If you double-click, or select the **Details** option on the context menu, the following screen is displayed:&nbsp;

![Image](<lib/batch-process-import-a-posted-batch-detail.png>)

7. All the details of the transactions entered or imported and posted (updated) to the ledger for the selected batch, will be displayed. Check that the selected batch (journal) contains the transactions which needs to be imported, and click on the **Batch entry** tab.&nbsp;

| ![Image](<lib/admon-note.png>) | *You may click on the **Print** button to print the transactions in the posted batch. An example of the "Batch type - Posted batch" report, is as follows:* ![Image](<lib/batch-process-import-a-posted-batch-printed.png>) |
| --- | --- |


| ![Image](<lib/admon-note.png>) | *Only those batches for the selected batch type will be listed.* |
| --- | --- |


8. Select a batch and click on the **OK** button. All the transactions in the selected, posted batch will be imported into the selected batch. An example, is as follows:

# ![Image](<lib/batch-process-import-a-posted-batch-imported.png>)

| ![Image](<lib/admon-note.png>) | *Should the processing icons on the "Batch entry" screen is inactive during this process, you may need to close and reopen the batch.*&nbsp; |
| --- | --- |


| ![Image](<lib/admon-note.png>) | *Once you have finished entering / editing the transactions in the batch you need to: -* [*Change the Alias (Batch name)*](<batch-change-alias.md>)*.* [*Balance the batch*](<batch-balance.md>)*.* [*List the batch*](<batch-list.md>)*.* [*Post the batch*](<batch-post.md>)*.* |
