# Parameters (Report options)

***

Parameters allows specific selections (choices) in how the data is printed and the data that need to be included or excluded from the report.

When creating a parameter, a [*Delphi Form*](<http://delphi.about.com/library/weekly/aa020299.htm> "target=\"\_blank\"") \*.dfm parameter file will be created with the same name as your report.&nbsp;

In the Report man screen, the default report icon will change once a parameter is added:&nbsp;

| **Icon - Before parameters is added** | **Icon - After parameters is added** |
| --- | --- |
| ![Image](<lib/reportman-report-icon-no-parameters.png>) | ![Image](<lib/reportman-report-icon-parameters.png>) |


Once parameters is added, the report will consist of two (2) files (the report file (\*.rep) and the parameter file (\*.dfm). If you need to copy a report you need to copy both files.&nbsp;

When you click on the **Refresh** button, the icon will change in the Report man screen. This indicates that the report consists of two (2) files (for example if parameters is added to the "*Copy of Empty.rep*" report):

* Copy of Empty.rep
* Copy of Empty.dfm

| ![Image](<lib/admon-note.png>) | *When copy the report to a folder in the File explorer, you need to copy both these files.* *When renaming the report in the File explorer, you need to rename both these files with the same name.* |
| --- | --- |


&nbsp;

In this example, the following parameters is documented:

1. [Sort sequences parameters](<reportman-parameters-sort.md>) - Allows to change the sequence in which the data is displayed on a report.
1. [Lookup parameters](<reportman-parameters-lookup.md>) - Allows to lookup a specific account or a range of accounts to include in the report.
1. [Groups parameters](<reportman-parameters-groups.md>) - Allows a list box in to select a group or groups to include in the report.
