# Point-of-Sales (POS)

The point-of-sales (POS) document layout file includes a versatile and dynamic option that caters to a wide range of language preferences and customisation needs. This translatable delivery note layout is powered by our language files feature, ensuring seamless translation of content across various languages. Not only does it adapt to the default language of your Set of Books, but it also accommodates debtor and creditor language preferences when they differ from the default.

## POS - START -SP100 - layout file

This specific layout file may be used for documents generated by point-of-sale printers with model SP100 (or a similar designation).

![Image](<lib/Layout-file-POS-STARTSP100.png>)

The "*POS - START -SP100"* layout file does prints stock codes and stock descriptions.&nbsp;

The Cashup in the footer sections will only display actual values when POS payments are processed.

# Document layout files supports additional languages for Debtors / Creditors

The translatable "*POS - STARTSP100*", "*POS 1*" and "*POS 2*" document layout file is a feature designed to accommodate multiple languages within osFinancials5/TurboCASH5. These files facilitate printing documents in either the chosen language or the default language associated with the Set of Books. Additionally, these translatable document layout files take into account language preferences specified for debtor (customer/client) or creditor (supplier/vendor) accounts, including international ones.

In osFinancials5/TurboCASH5, users have the flexibility to designate a particular language for debtor or creditor accounts:

* **Debtors**: When a specific language is assigned to a debtor account, osFinancials5/TurboCASH5 will print document layout files for sales-related documents (such as invoices, credit notes, quotes, and point-of-sale invoices) in the debtor's preferred language.&nbsp;
* **Creditors**: If a specific language is set for a creditor account, osFinancials5/TurboCASH5 will print document layout files for purchase-related documents (including purchase orders, supplier returns, and orders) in the preferred language of the creditor, whether they are domestic or international.

![Image](<lib/language-debtor-creditor.png>)

This feature ensures that financial documents align with the language preferences of both customers and suppliers, enhancing communication and clarity for a diverse range of business interactions and in regions with multiple languages. For instance, in a country like South Africa, documents can be generated in languages like Afrikaans or English.
