# Reportman - Two pass report setting

***

The Two pass report setting allows you to preview the first page before all the pages are calculated (e.g. page 1/2, etc.). This is the default setting implemented in reports and document layout files which supports multi-page and subreports in reports and document layout files. This controls the print speed and memory usage (when printing, pages printed are released from memory).

| ![Image](<lib/admon-note.png>) | *This two pass report setting must be selected (ticked) for both, PAGECOUNT and/or SkipToPage settings.* |
| --- | --- |


&nbsp;

Your report or document layout file will produce the following message:&nbsp;

*“Two pass report needed-Page setup”*&nbsp;

Click **OK**.&nbsp;

If this error is encountered in the "*Report manager designer*" screen, go to **File → Page setup** menu and on the **Print setup** tab, tick the "*Two pass*" report field.

![Image](<lib/reportman-twp-pass-report-setting.png>)

Click **OK**.

| ![Image](<lib/admon-note.png>) | *If this error is encountered when printing the report on screen, click on the* ![Image](<lib/report-preview-icon-setup.png>) *"Show report page setup" icon or press the **F11** key.* *On the **Print setup** tab, tick the "Two pass report" field.* *Click **OK**.* |
