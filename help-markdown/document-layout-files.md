# Preview of available document layout file samples

To assist in choosing a layout file for your needs, you may preview some screenshots of document layout files:

1. [**Translatable Document Layout Files with Dynamic Document Types**](<document-layout-files-translate.md>):
- &nbsp;

  - *"Layout File"*
  - *"Document Layout"*
  - *"Document Layout (15 Code)"*

These layout files utilize translatable expressions facilitated by language files and osFinancials5/TurboCASH5 settings. They dynamically display the appropriate document type names such as "*Invoice," "Credit note," "Quote," "Purchase," "Supplier return,*" or "*Order*" based on the specific document being printed. The document types are sourced from the *"TYPES.SDESCRIPTION*" expressions within the layout.

2. [**Translatable Document Layout Files with Dynamic Document Types without VAT/GST/Sales Tax**](<document-layout-files-no-tax.md>):
- &nbsp;

  - *"NO-TAX Layout File"*
  - *"NO-TAX Document Layout"*
  - *"NO-TAX Document Layout (15 code)"*

If you are not registered for VAT/GST/Sales tax or have chosen not to opt for voluntary registration, these layout files exclude VAT/GST/Sales tax information. They also dynamically display the appropriate document type names based on the specific type of document being printed.

These layout files utilize translatable expressions facilitated by language files and osFinancials5/TurboCASH5 settings. They dynamically display the appropriate document type names such as "*Invoice," "Credit note," "Quote," "Purchase," "Supplier return,"* or *"Order"* based on the specific document being printed. The document types are sourced from the "*TYPES.SDESCRIPTION*" expressions within the layout.

3. [**Document Layout Files in English with Static Text**](<document-layout-files-english.md>):
- &nbsp;

  - *"TAX Invoice - English"*
  - *"NO-TAX Invoice - English"*

These layout files offer a simple design featuring fixed text labels in the English language. Unlike translatable expressions, these layouts utilize static text labels. They are well-suited for customization to suit a range of document types, including *"Invoice", "Credit note", "Quote", "Purchase", "Supplier return",* or *"Order".* You have the flexibility to personalize these files according to your specific layout requirements and English translations. Additionally, you can leverage the Invoice layout as a foundation for creating other document types.

4. [**Translatable Delivery Note Document Layout File**](<document-layout-delivery-note.md>)**:** The delivery note document layout file includes a versatile and dynamic option that caters to a wide range of language preferences and customisation needs. This translatable delivery note layout is powered by our language files feature, ensuring seamless translation of content across various languages. Not only does it adapt to the default language of your Set of Books, but it also accommodates debtor and creditor language preferences when they differ from the default.
4. [**Translatable Document Layout Files for Point-of-Sale (POS)**](<document-layout-pos.md>): These layout files are designed specifically for point-of-sale systems, catering to the needs of retail transactions at the point of sale.
4. [**Document Layout Files in Dutch Language**](<document-layout-files-dutch.md>): These layout files are previewed in the Dutch language, providing options for document layout customization for users in the Netherlands.

By offering these different layout file samples, osFinancials5/TurboCASH5 provides users with a range of choices to accommodate their needs, whether it's for different document types, languages, VAT/GST considerations, or point-of-sale transactions. This variety allows users to select the most suitable document layout template for their specific requirements.
