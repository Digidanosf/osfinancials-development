## Ledger

The [general ledger](<ledger-accounts.md>) is a core component of the accounting system in osFinancials5/TurboCASH5. It serves as a central repository for recording and storing all financial transactions and budgets of a business. The general ledger contains individual accounts that track various types of financial activities, such as revenue, expenses, assets, liabilities, and equity.

Each account in the general ledger represents a specific category or type of transaction, such as sales, purchases, expenses, etc. Transactions are recorded in the general ledger using a double-entry bookkeeping system, where every transaction has an equal debit and credit entry, ensuring that the accounting equation remains in balance.

*Assets = Liabilities + Equity*&nbsp;

The general ledger provides a detailed and chronological record of all financial activities, allowing businesses to track and analyse their financial performance. It serves as the foundation for generating important financial reports such as the trial balance, income statement, and balance sheet.

In osFinancials5/TurboCASH5, the general ledger module allows users to create and manage their chart of accounts, record transactions, manage budgets, generate financial reports, and perform other essential accounting tasks. It provides a comprehensive view of a company's financial position and financial performance. This helps to facilitate accurate and reliable financial management and decision-making.

By effectively utilizing open item processing in osFinancials5/TurboCASH5, businesses can streamline their reconciliation processes, ensure accuracy in financial records, track and resolve outstanding transactions efficiently, and maintain up-to-date and reliable financial information.

## Small backup&nbsp;

## Uni dump&nbsp;

If your Set of Books in osFinancials5/TurboCASH5 is created as a Microsoft SQL Server (MSSQL) database type, the "uni dump" file created as an SQL file type can be used to restore the Set of Books.

The "*uni dump*" file contains SQL statements that represent the structure and data of the database. By executing these SQL statements, you can recreate the database and restore it to its previous state.

To restore a Set of Books from a "*uni dump*" file in osFinancials using MSSQL, you would typically follow these steps:

1. Ensure that you have a compatible version of MSSQL installed and running.
1. Create an empty database in MSSQL that will serve as the destination for the restored Set of Books.
1. Open the "*uni dump*" file in a text editor or an SQL client.
1. Execute the SQL statements from the "uni dump" file in the MSSQL database. This can be done using an SQL client or by running a script.
1. Once the SQL statements have been executed successfully, the Set of Books should be restored in the MSSQL database.

It's important to note that the specific steps and tools for restoring the database from a "*uni dump*" file may vary depending on your environment and the version of MSSQL you are using. It's recommended to consult the documentation or resources specific to your version of MSSQL for accurate instructions on restoring a database from an SQL file. Additionally, it's always a good practice to backup your database before performing any restoration to ensure data integrity.

# Document E-mail Pro plugin

The [Document E-mail Pro plugin](<plugin-document-mailpro.md>) in osFinancials5/TurboCASH5 allows you to send documents by email directly from the program. This plugin is included in the osFinancials5 Business class subscription or can be purchased separately with a once-off licence.

With the Document Email Pro plugin, you can easily send documents such as invoices, quotes, credit notes, and order lists to customers or wholesalers. Different email templates can be created to match the specific document being sent. For example, the email text sent to a customer with an invoice will differ from the text sent to a wholesaler with an order list.

The Document E-mail Pro plugin can also be used in conjunction with other plugins like [remittance](<https://www.osfinancials.org/en/webshop/debtor-creditor-control/remittance-plugin> "target=\"\_blank\""), [subscriptions](<https://www.osfinancials.org/en/webshop/branch-specific-solutions/membership-subscription-plugin> "target=\"\_blank\"") and [newsletter](<https://www.osfinancials.org/en/webshop/email/document-email-pro-plugin-newsletter> "target=\"\_blank\"").

When sending documents as email attachments, you have the option to choose different file formats, including PDF and UBL 2.1. Here's an explanation of each format:

1. **PDF (Portable Document Format):** PDF is a widely used file format that preserves the formatting, fonts, and layout of a document. It ensures that the recipient sees the document exactly as intended, regardless of their software or operating system. PDF files are typically read-only, allowing recipients to view and print the document but not easily edit its content. Sending documents as PDF attachments guarantees compatibility and easy viewing for recipients.
1. **UBL 2.1 (Universal Business Language)**: UBL is an XML-based standard for exchanging business documents electronically. UBL 2.1 is a specific version of the UBL standard. UBL files follow a predefined structure and contain structured information about various business documents such as invoices, purchase orders, and payment instructions. UBL files are commonly used in business-to-business (B2B) transactions to facilitate automated processing and interoperability between different software systems. By sending documents as UBL 2.1 attachments, you provide recipients with machine-readable data that can be easily processed by their business systems.

When emailing documents, you can attach both a PDF file for human-readable viewing and a UBL 2.1 file for machine-readable data. This offers flexibility to the recipient in interacting with the information based on their specific needs and systems.
