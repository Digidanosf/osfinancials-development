# Copy an existing Set of Books

***

This option allows you to create a new Set of Books by making an exact copy of an existing one, including all the data it contains. You can select the Set of Books you want to copy, and the system will create a new one in a directory of your choice.

This new Set of Books can be accessed within the osFinancials5/TurboCASH5 program, even if it is not created in the osFinancials5/TurboCASH5 directory. You may keep your Set of Books in any mapped directory, i.e. on a local or network drive.&nbsp;

**To create a copy of a Set of Books:**&nbsp;

1. On the **Start** ribbon, you can select the **Create Set of Books** option or simply press the **F10** key on your keyboard.

![Image](<lib/books-create-copy-books.png>)

2. osFinancials5/TurboCASH5 automatically defaults to the option “*Let osFinancials help you to create a Set of Books?*”, as this is the recommended option for most users. Select the "*Copy an existing Set of Books?"* option.&nbsp;
2. Click on the **Next** button.\
![Image](<lib/books-create-copy-books-select.png>)
2. Select any of the available Set of Books on your system. If you do not find the Set of Books on the list, you may click on the **Browse** button to find a Set of Books from the "*Pick a directory*" screen.&nbsp;
2. Click on the **Next** button. &nbsp;

![Image](<lib/books-create-copy-books-enter-name.png>)

6. Enter the name for your new Set of Books.&nbsp;

| ![Image](<lib/admon-important.png>) | *The name must not already exist and may not be longer than 50 characters. This name will help you to identify your Set of Books on your system and you will need to select this name, whenever you need to open the Set of Books or perform some other functions on the Set of Books.*&nbsp; |
| --- | --- |


| ![Image](<lib/admon-note.png>) | *The Directory in which you create the Set of Books is displayed underneath the Location **Browse** button. It will usually be the default directory in which osFinancials5/TurboCASH5 is installed, or the directory&nbsp; which you have specified when you installed* osFinancials5/TurboCASH5#8202;*. If you wish to create your Set of Books in any other directory or location on your system, you may click on the **Browse** button to select any folder or drive on the "Pick a directory" screen.*&nbsp; |
| --- | --- |


7. Click on the **Finish** button.&nbsp; A confirmation message is displayed:&nbsp;

*&nbsp;Are you sure you want to create the MY-BOOKS Set of Books?*&nbsp;

8. To confirm that you want to create the Set of Books, click on the **Yes** button. An information message

*&nbsp;MY-BOOKS Set of Books created\!*

9. Click **OK**. The new Set of Books, which is a copy of another Set of Books, is automatically opened and ready to work in.

| ![Image](<lib/admon-note.png>) | ***Year-end confirmation***&nbsp; *Should you open the Set of Books, and the date of the end of the year, is in the past, a confirmation message will be displayed, for example;*&nbsp; *This year Ends on 2022/02/28* *Today 2022/04/04* *Warning, this date does not fall into any of this or last year's reporting dates\!* *Do year end?* *You need to&nbsp; adjust the Start of the financial year in one of the following options on the Setup ribbon:* &nbsp; *Setup **→ [**Reporting dates*](<setup-reporting-dates.md>) *(if no transactions were processed for the Set of Books).*&nbsp; [*Do year end*](<do-year-end.md>)*.*&nbsp; |
| --- | --- |


| ![Image](<lib/admon-important.png>) | ***If you have created a copy of an existing Set of Books, which already has transactions and balances:*** *If you do wish to have a clean Set of Books with no transactions and balances, you may need to clear / reset the transactions and balances in the Set of Books. (See **Global processes → [Clear / Reset***](<clear-reset-set-of-books.md>) *on the [**Setup***](<setup-ribbon.md>) *ribbon). All the master files (accounts, etc.) will be available and you may then enter your own transactions and balances in the new Set of Books.*&nbsp; |
| --- | --- |


| ![Image](<lib/admon-tip.png>) | *This is similar to&nbsp; [**Save as***](<books-save-as.md>) *option on the [**Start***](<start-ribbon.md>) *ribbon), where you may save an exact copy of an existing Set of Books with a new name.*&nbsp; |
| --- | --- |


| ![Image](<lib/admon-tip.png>) | *You may also copy all the settings and data of an existing Set of Books into a new Set of Books using **Global processes → [Copy from a Set of Books***](<copy-from-set-of-books.md>) *on the [**Setup***](<setup-ribbon.md>) *ribbon.*&nbsp; |
