# MSSQL Server configuration, settings and Firewall settings to open a Set of Books

When opening an MSSQL database type Set of Books in osFinancials5/TurboCASH5, there are certain server, account control, and firewall settings that need to be configured correctly. These settings ensure that osFinancials5/TurboCASH5 can establish a connection to the MSSQL Server and access the database.

Here are some considerations regarding the server, account control, and firewall settings:

1. **MSSQL Server Configuration**: The MSSQL Server needs to be properly installed and configured on the server machine. This includes ensuring that the server is running, the necessary services are active, and the database is accessible.
1. **Server Name and Credentials**: When opening an MSSQL database type Set of Books, you need to provide the correct server name or IP address along with the appropriate credentials (username and password) to establish a connection to the MSSQL Server. These details should be provided during the Set of Books setup process.
1. **Account Control Settings**: Depending on the security settings of your MSSQL Server, you may need to ensure that the user account used to access the database has the necessary permissions and privileges. This includes granting the account appropriate access rights to the database.
1. **Firewall Settings**: If your server machine has a firewall enabled, you may need to configure the firewall to allow incoming connections to the MSSQL Server. This involves opening the necessary ports (e.g., port 1433 for MSSQL) to allow communication between&nbsp; osFinancials5/TurboCASH5 and the database server.

It's important to consult with your IT administrator or database administrator to ensure that the server, account control, and firewall settings are properly configured to enable the opening of an MSSQL database type Set of Books in osFinancials5/TurboCASH5.
