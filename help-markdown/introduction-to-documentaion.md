# Differences in osFinancials5 and TurboCASH5&nbsp;

The documentation provided for both osFinancials5 and TurboCASH5 is designed to be comprehensive and applicable to both software versions, despite their minor differences. Here’s a summary of the key points and differences:

**Shared Source and Primary Differences**

* **Shared Source**: Both osFinancials5 and TurboCASH5 share the same source code, which means they have a lot of common features and functionalities.
* **Primary Difference**: The primary difference between the two is the base, which influences some of the interface and address formats.

**Address Layout Differences**

* **osFinancials5**: The address format is displayed as:

  * Address 1, Postal code, Address 2, Address 3.

* **TurboCASH5**: The address format is displayed as:

  * Address 1, Address 2, Address 3, and Postal code.

**Business Class Subscription**

* **Basic Subscription**: Both versions include processing of more than 500 transactions and some plugins are included in the subscription.
* **Plugins**: Some plugins are designed specifically for the Dutch market and are not available in TurboCASH5. Plugins designed for the international market are included in both osFinancials5 and TurboCASH5.

**Documentation Formats**

* **EngHelp.chm File**: This context-specific help file can be launched using the **F1** key from most screens in the osFinancials5 and TurboCASH5 interfaces.
* **Online Help**: Available in [English Online help](<https://www.osfinancials.org/htmlhelp\_eng/welcome.html> "target=\"\_blank\"") and [Afrikaans Online help](<https://www.osfinancials.org/htmlhelp\_af/welcome.html> "target=\"\_blank\""), This online help will be updated later and may include additional or updated help topics. This is supplementary to the EngHelp.chm file included in the installations.

**Applicability**

* **Documentation Coverage**: The documentation covers all features in both osFinancials5 and TurboCASH5, ensuring that users of either software can find the information they need.
* **Screens and Text**: While the documentation may refer to osFinancials5 in screenshots and text, the information is generally applicable to TurboCASH5 as well.

**Conclusion**

The documentation is designed to be flexible and comprehensive, catering to users of both osFinancials5 and TurboCASH5. The minor differences in address formats and available plugins are noted, but the core functionalities and features are shared between the two software versions.

# Plugins in osFinancials5 and TurboCASH5

**Plugins Specific to TurboCASH5**

* **Plugins Set to False**: Some plugins are specifically designed for the Dutch market and are set to false in TurboCASH5. These plugins are not available in the TurboCASH5 subscription.
* **Plugins Menu**: The Plugins menu in TurboCASH5 will only list the plugins that are shipped and included with the TurboCASH5 subscription.

**Included Plugins in osFinancials5 Subscription / TurboCASH5 Subscription**

* **EmailInvoice Document E-mail Pro**: Allows for the emailing of invoices directly from the software.
* **Bank Import**: Enabled in **Setup → Data parameters** by ensuring the "*Default bank reconciliation method*" field is not ticked.

**Marketable Plugins**

These plugins are functional and can be used with both osFinancials5 and TurboCASH5. Some may require additional licenses.

* [**Activa**](<activa-plugin.md>): Plugin for managing assets.
* [**Charter**](<plugin-charter.md>): Specific functionality for generating charts.
* [**Document Export Import**](<plugin-document-export-import.md>): Allows for the export and import of documents.
* [**DrillDown**](<plugin-drill-down.md>): Enables drill-down functionality for detailed data analysis.
* [**Freefields**](<plugin-free-fields.md>): Adds customisation of fields to various forms.
* [**LocationBatchAndSerials**](<plugin-serials-batches-location.md>): Manages locations, batches, and serial numbers.
* [**Multimedia**](<plugin-multimedia.md>): Integrates multimedia elements into the software.
* [**Multi-user Agenda**](<plugin-multi-user-agenda.md>): Provides a shared agenda for multiple users.
* [**Remittance**](<plugin-remittance.md>): Facilitates remittance processes.
* [**Price agreement**](<plugin-price-agreement.md>): Manages pricing agreements.
* [**Subscriptions**](<plugin-subscriptions.md>): Handles subscription-based billing.

**Essential Plugins**

These plugins are crucial for specific functionalities within osFinancials5 and TurboCASH5.

* **ReportMan - [Report designer**](<reportman.md>): Essential for customising layout files and reports.
* [**Register plugins**](<plugin-register.md>): Adds licenses to plugins.
* [**SQL Browser**](<plugin-sql-browser.md>): Essential for adding reporting groups when [Customising your own Set of Books](<create-books-advanced.md>).
* **HandyTools**: Advanced tools for experienced users.

**Plugins That Cannot Be Disabled**

* **DYNADBREG**: This plugin cannot not be disabled and is essential for certain functionalities within osFinancials5/TurboCASH5.

**Summary**

The plugins available in TurboCASH5 are tailored to meet the needs of users while excluding those specific to the Dutch market. Both osFinancials5 and TurboCASH5, includes Essential plugins like Report designer, SQL Browser, and HandyTools are included to ensure core functionalities are maintained. Marketable plugins offer additional features that can be licensed as needed, enhancing the osFinancials5 and TurboCASH5 capabilities.
