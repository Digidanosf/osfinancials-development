# Context menu - Documents

***

The "*Documents*" context menu option on the "*Document list*" screen changes for the document type selected.&nbsp;

Depending on whether a posted document; or an unposted document is selected; some context menu options may not be available, or additional specific context menu options may be added.&nbsp;

Some examples are:

[Change account](<docs-grid-change-account.md>) - is not available for posted documents.&nbsp;

[**Adjustment**](<documents-adjustment-osf4.md>) - is not available for unposted documents and is not implemented in osFinancials5/TurboCASH5.

# Sales documents

## Invoices - Documents context menu

&nbsp;

![Image](<lib/invoices-context-menu-documents.png>)

&nbsp;

The options is as follows:

1. [**Copy document**](<document-grid-menu-copy-document.md>) - Create an exact copy of a selected document and add it as a new document to the list for the same document type.
1. [**Create purchase on default supplier 1**](<doc-grid-purchase-def-supplier.md>) – This option allows you to create an order for a default preferred supplier 1 (as selected for a specific stock item (**Stock items** on the **Default** ribbon)).
1. [**Convert invoice to order**](<doc-grid-convert-invoice-order.md>) - Create an Order from a selected Invoice or Quote. The&nbsp; "*Creditor accounts*" screen will be displayed on which you need to select a creditor (supplier / vendor) account. This will add the converted Invoice or Quote as a new document to the list of Orders. You may edit the Order and then confirm (convert) the Order to a Purchase document.
1. [**Copy to**](<document-grid-menu-copy-to.md>) - In addition to the **Documents → [Copy document**](<document-grid-menu-copy-document.md>) option on the context menu of the "*Document list*" screen, to create exact copies of the selected document(s) of a specific document type, you may also copy any of the documents to another document type. &nbsp;
1. [**Convert to credit note / supplier return**](<doc-grid-convert-invoice-credit.md>) **-** Convert selected Invoice(s) to Credit note(s).&nbsp;
1. [**Redo discount on document**](<docs-grid-redo-discount.md>) **-** This will refresh the discount on a selected document.
1. [**Change account**](<docs-grid-change-account.md>) - Select another debtor (customer / client) account for a selected unposted Invoice.&nbsp;
1. [**Edit account**](<docs-grid-edit-account.md>) - Edit the debtor (customer / client) account for a selected document.
1. [**Create backorders to deliver**](<documents-create-backorders.md>) - If an outstanding backorder exists for a debtor (customer / client), you may use this option to create a new Invoice for the outstanding backorders to deliver.&nbsp;
1. [**Adjustment**](<documents-adjustment-osf4.md>) - This option is only available for posted documents. This option to edit the document header of a posted document is not implemented in osFinancials5/TurboCASH5.&nbsp;
1. [**Create account from document details**](<docs-grid-create-account-doc-det.md>) - This document context menu allows you to create a new debtor (customer / client) account for unposted Invoices, using the document details.&nbsp;
1. [**Confirm**](<docs-grid-change-status.md>) - By default, the "*Status*" of an unposted document (i.e. Invoice, Credit note, Purchase or Supplier return), is set as "*Unposted*". Unposted documents can be edited. You may change the "*Status*" of unposted documents (i.e. Invoices, Credit notes, Purchases or Supplier returns),&nbsp; from "*Open*" to "*Confirmed*".&nbsp; Once document status, is set to "*Confirmed*" you may filter the document status for a Confirmed documents. Posting "*Confirmed*" unposted documents (i.e. Invoices, Credit notes, Purchases or Supplier returns), will change the document's "*Status*" from "*Confirmed*" to "*Posted*".
1. [**Spreadsheet import**](<docs-grid-spreadsheet-import.md>) - &nbsp;
