# Open item link - Due tab

***

The **Due** tab will list all transactions for all open item accounts (i.e. ledger, debtors and creditors), which is set as open item accounts. &nbsp;

The transaction or outstanding transaction amounts are added in columns based on the number of days (0-29 days, 30-59 days, 60-89 days and +90 days).&nbsp;

You may to view and generate the report for a selected date options:

* **Date of ageing** - Select the date or due date option. The default is "*Date of ageing*" is the "*Current date*" (which is the date of your operating system). &nbsp; This report will include all outstanding transactions of only the open item accounts, for selected account types.&nbsp;
* **Date of ageing** - Select a specific date of the ageing to include in the report.&nbsp; This report will include all outstanding transactions of all accounts, including those non-open item accounts,&nbsp; for selected account types.&nbsp;

# Date of ageing - Current date

**To generate and print ageing report for the current date:**

1. Select **Open item link** (**Default** ribbon).&nbsp;
1. Click on **Due** tab.

![Image](<lib/open-item-age-analysis-due-tab.png>)

3. Select the **account type** (i.e. ledger, debtors and creditors) to generate the **Age analysis** report for a specific account type.&nbsp;
3. **Date of ageing** - Select the date or due date option. The default is "*Date of ageing*" is the "*Current date*" (which is the date of your operating system). &nbsp; &nbsp;
3. Click on the **Print** button to print the "*opendays*" report.

![Image](<lib/open-item-ageing-due-report-current-date-prin.png>)

| ![Image](<lib/admon-note.png>) | *&nbsp;Only the transactions of accounts set as open item accounts will be included in this report.* |
| --- | --- |


# Date of ageing - Selected date

**To generate and print ageing report for a selected date:**

1. Select **Open item link** (**Default** ribbon).&nbsp;
1. Click on **Due** tab.

![Image](<lib/open-item-ageing-due-report-date.png>)

3. Select the **account type** (i.e. ledger, debtors and creditors) to generate the **Age analysis** report for a specific account type.&nbsp;
3. **Date of ageing** - The default is "*Date of ageing*" is the "*Current date*" (which is the date of your operating system).&nbsp; Select a specific date of the ageing.&nbsp; &nbsp;

| ![Image](<lib/admon-note.png>) | *In this example, the date of ageing is set to 30 April 2021.*&nbsp; *"Correction transaction ageing COR" will automatically be generated to include the outstanding transactions of all accounts for selected account types.*&nbsp; *The "Correction transaction ageing COR" message, is only displayed for transactions of non-open item accounts.*&nbsp; |
| --- | --- |


5. Click on the **Print** button to print the "*opendays*" report.

In this example, the date of ageing is set to 30 April 2021.&nbsp;
