    Access control - Passwords

    In osFinancials, you have the option to create and manage passwords for user authentication in the Setup â†’ Access control menu on the Setup ribbon. 

    It is recommended to create at least two users with passwords, with the main user being the administrator who has access to all default functions within a Set of Books.

    For additional users and any other users for whom you have created passwords (excluding the main user who is the administrator), you can utilise the User tab ribbon to customize the Set of Books in osFinancials. You can also review and configure specific settings to personalise the use of certain functions and preferences on the Standard security, Reports access, Batches access, Others, Documents, Groups and Usertabs tabs.

    In addition to the options provided on these tabs for customising functions and preferences, you can also restrict specific users from accessing or performing certain functions within osFinancials Set of Books.

    To ensure security, it is advisable to change passwords periodically. The password field consists of eight characters, including alphanumeric and special characters. It is important to note that passwords are case-sensitive. When a user password is set, it becomes essential to enter the password correctly when performing actions such as opening or deleting a Set of Books, or executing specific functions like year-end processes.

    When accessing a Set of Books, deleting a Set of Books, or performing certain functions, a login screen will prompt you to enter a valid username (if required) along with the corresponding password associated with that username.

    In addition to setting passwords for osFinancials Set of Books, it is also recommended to implement user control and passwords for your operating system. This provides an additional layer of security for accessing the osFinancials software and helps protect sensitive financial information. By using user control and passwords for the operating system, you can ensure that only authorized individuals can access and use the software and its associated data.

    In addition to the access control feature in osFinancials, you have the option to set and configure passwords for your operating system and/or network. This refers to the passwords used to secure access to your computer or network infrastructure, separate from the application itself.

    Setting strong and unique passwords for your operating system and network is an essential security measure to protect your osFinancials data and prevent unauthorized access. By implementing password policies and configurations at the operating system and network levels, you add an extra layer of security to your overall system.

    Here are some considerations for setting and configuring passwords for your operating system and network:

    1. Strong Passwords: Choose passwords that are complex, combining uppercase and lowercase letters, numbers, and special characters. Avoid using easily guessable or common passwords.
    2. Unique Passwords: Use different passwords for different accounts or systems. Reusing passwords across multiple platforms increases the risk of unauthorized access if one password is compromised.
    3. Password Length and Complexity: Set minimum password length requirements and enforce complexity rules to ensure passwords are sufficiently robust.
    4. Password Expiration and Renewal: Implement a password expiration policy that requires users to change their passwords at regular intervals. This practice helps mitigate the risk of password-related vulnerabilities.
    5. Account Lockout Policies: Configure account lockout settings to automatically lock user accounts after a certain number of failed login attempts. This helps prevent brute-force attacks and unauthorized access attempts.
    6. Two-Factor Authentication (2FA): Consider implementing two-factor authentication for your operating system or network, which adds an extra layer of security by requiring users to provide a second form of verification, such as a code sent to their mobile device.

    By ensuring strong and unique passwords, enforcing password policies, and implementing additional security measures like two-factor authentication, you can strengthen the overall security of your operating system and network. This, in turn, helps protect your osFinancials data and reduces the risk of unauthorized access or data breaches.
