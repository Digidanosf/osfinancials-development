    FIXED - Due Ageing

    Added the 3 old age analysis reports in TurboCASH 5.1 and osFinancials 4 to User reports → Debtors and Creditors menu

    The reason for this is to be in sync with osFinancials 5 on Git - GIT-osF-Source.git

    The Help → Update option when a stable release of osFinancials is updated in TurboCASH5.2 it does not clash with osFinancials source in the "system reports" folder.  

    Age analysis reports osFinancials 4 / TurboCASH5.1 vs TurboCASH5.2  

    As from TurboCASH5.2 / osFinancials5 – the age analysis reports were changed as follows: 

    • Age analysis reports which were on the Reports → Debtors and Reports → Creditors menu, have been dropped. 
    • The Age analysis on the Reports ribbon was introduced. This age analysis report lists all non-open-item accounts; if the Date of ageing = “Current date” is changed to a selected date. 

    Discontinued Debtor / Creditor ageing reports in osFinancials4 added in TurboCASH5.2

    Since osFinancials / TurboCASH4, TurboCASH5.1 / osFinancials4 the age analysis reports were included in all releases.

    Date of ageing = Current date lists only the Accounts set as open item:

    Date of ageing = Current date lists only the Accounts set as open item:

     

    Date of ageing = Selected date - e.g. System date = 2020/04/01 and selected date of ageing = 2020/04/01 lists all the Accounts set as open item as well as non open item accounts. 

    The example in this database; Debtors and Creditors - set as Open item on the “Accounting information” tab.:  

    • Debtors - only two debtor accounts (i.e. “AB Green” and “Mike Davies”) and 
    • Creditors – only two creditor accounts (i.e. “ABC Office Supplies” and “A-One Suppliers”), 

    For the non open item accounts, it adds COR in the description and Corrections with the Amount.

    Selecting a date 

    In this database no Ledger accounts is set to Open item. If the Ledger option is selected, and selected date of ageing = 2020/04/01, it lists all ledger account balances
