    Customizing Document Type Terminology - Quote

    The language associated with each Set of Books determines the specific labels used to cater to different countries or tax regulations. In most languages, the term 'Quote' or its equivalent will be displayed. For instance, en-UK and en-South Africa use terminology related to 'Quote', while in South Africa, Afrikaans refers to 'Kwotasie' or 'Kwotasies'. The United States and Canada employ the term 'Estimates'.

    These quote-related labels are visible within the osFinancials interface when you access a Set of Books. They are also present in reports and document layout files generated by the system.

    To adapt the quote terminology to suit your country, jurisdiction, and industry, you can customize and modify the quote-related labels via the "Customize Languages" screen. This screen is accessible through the Tools â†’ Customise Language menu on the Setup ribbon.

    You may also find it helpful to utilize the search options available. This enables you to search for a specific label either in the currently active language file you're editing or in the default "English" language file. This assists you in identifying and modifying specific text to better suit your requirements.

    Document type names for layout files that support all document types

    osFinancials supports a single document layout file which utilizes translatable expressions facilitated by language file settings. They dynamically display the appropriate document type names such as "Invoice," "Credit note," "Quote," "Purchase," "Supplier return," or "Order" based on the specific document being printed. The document types are sourced from the "TYPES.SDESCRIPTION" expressions within the layout. 

    To change the document type names, you need to change the descriptions on the Types tab of the  "Customize Languages" screen (accessible from the Tools â†’ Customise Language menu on the Setup ribbon).
