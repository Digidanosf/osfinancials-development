    Customizing Document Type Terminology - Invoice

    The language associated with each Set of Books determines the specific labels used to cater to different countries or tax regulations. In most languages, the term 'Invoice' or its equivalent will be displayed. For instance, en-UK and en-South Africa use terminology related to 'Invoice,' while in South Africa, Afrikaans refers to 'Faktuur' or 'Fakture'.

    These invoice-related labels are visible within the osFinancials interface when you access a Set of Books. They are also present in reports and document layout files generated by the system.

    To adapt the invoice terminology to suit your country, jurisdiction, and industry, you can customize and modify the invoice-related labels via the "Customize Languages" screen. This screen is accessible through the Tools â†’ Customise Language menu on the Setup ribbon.

    You may also find it helpful to utilize the search options available. This enables you to search for a specific label either in the currently active language file you're editing or in the default "English" language file. This assists you in identifying and modifying specific text to better suit your requirements.

    Document type names for layout files that support all document types

    osFinancials supports a single document layout file which utilizes translatable expressions facilitated by language file settings. They dynamically display the appropriate document type names such as "Invoice," "Credit note," "Quote," "Purchase," "Supplier return," or "Order" based on the specific document being printed. The document types are sourced from the "TYPES.SDESCRIPTION" expressions within the layout. 

    To change the document type names, you need to change the descriptions on the Types tab of the  "Customize Languages" screen (accessible from the Tools â†’ Customise Language menu on the Setup ribbon).

    If '%s ' is included in a label description in the language file, please refrain from translating this. This placeholder is reserved for specific data that will be dynamically populated within the label. For example, 'Cur : %s Quote : %s Order : %s Exp : %s' signifies that the current, quote, order, and expected quantities will be inserted and displayed within the program.

    If you come across "<@APPNAME@>" within a label description, be aware that it will be replaced with "osFinancials" in the program's interface. In your customized language file, which you are currently editing, you have the option to substitute "<@APPNAME@>" with "osFinancials" to accurately reflect the application's name within the interface.
