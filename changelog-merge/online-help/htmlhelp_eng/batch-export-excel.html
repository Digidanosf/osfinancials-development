    Batch transactions - Spreadsheet Export

    You may export unposted batch transactions in a selected batch and save it in a spreadsheet "Excel Workbook (*.xlsx) file type.  

    You may then import it into a batch, when required. 

    To export batch transactions from a Excel Workbook (xlsx) file: 

    1. On the Default ribbon, select Batch entry (F2). 
    2. Select an unposted batch, and click Open. 

     

    1. Click on the Spreadsheet import icon. 
    2. The "Data Import Export" be launched without any data.

    1. On the "Data Import Export" screen, click Export. All the transactions (excluding the balancing transactions, if the batch is already balanced) in the unposted batch will be imported  the "Data Import Export" screen.  

    If the Batch includes balancing transactions, these will not be included in the "Data Import Export" screen.

    Imported or copied text in osFinancials screens from external files - Description lines in batches 

    When importing or copying text for account. group descriptions or any other screens, or forms, the fields from external programs such as spreadsheet, word documents, etc. the hyphenation   character "–" may be displayed as a pipe character " | " in lookups and some screens or  as "  � " in some screens and in reports, document layout files, etc.

    For example, Motor vehicles � Net value if copied or imported into Setup - Accounts as Motor vehicles – Net value (with the hyphenation format) it will be displayed as Motor vehicles � Net value on some screens and in reports and as  Motor vehicles | Net value on lookup screens.

    You need to use the dash sign "-" to be displayed correctly. 

    Solution:

    To ensure hyphens display correctly in osFinancials, use the standard dash character ("-") instead of the hyphenation character when copying or importing text from external files.
