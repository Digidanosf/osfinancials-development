    Known issues - Calendar - Contacts - Resource

    Calendar - Contacts - Add resource name

    Calendar - Add resource name. When accessing the Contacts in the Calendar for the first time in the "CUSTOMISE-EMPTY-BOOKS", (or using the Customise your own Set of Books (Advanced)? option (for Firebird database type or the MSSQL database type), no contacts will be listed. 

    To add the existing contacts:

    Right-click on the Contacts and select "Add contact..." on the context menu.  

    A confirmation message will be displayed:

    No resource have been defined! Would you like to add one now? 

    Click Yes to add a resource.  

    On the "Select a Resource" screen, Click Add New Resource. Selecting and editing an existing resource from the list will not add the contacts. 

    Enter a new resource name the contacts will be listed.  When adding the new resource name, somewhere the Calendar appointments will be blank.

    Deleting the new resource name for Contacts in "RESOURCES" table (SQL Browser) and exit osFinancials and reopen the Set of Books, the Calendar appointments will be listed. and the Contacts will be empty again.
